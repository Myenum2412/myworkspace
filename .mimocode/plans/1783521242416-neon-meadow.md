# Google Calendar & Outlook Integration Plan

## Overview
Connect Google Calendar and Microsoft Outlook Calendar to MyWorkSpace, allowing users to view external calendar events alongside tasks in the upcoming tasks calendar.

## Architecture

### Data Flow
```
User clicks "Connect Google/Outlook" 
  → Redirect to OAuth provider (Google/Microsoft)
  → Provider redirects back to callback URL
  → API route exchanges code for tokens
  → Tokens stored in MongoDB `calendar_connections` collection
  → Calendar events fetched and displayed in the calendar UI
```

### New MongoDB Collection
```
calendar_connections:
{
  _id, userId, orgId,
  provider: "google" | "microsoft",
  accessToken, refreshToken, tokenExpiry,
  calendarEmail, calendarName,
  syncEnabled: boolean,
  lastSyncAt, createdAt, updatedAt
}
```

### 3. Google Calendar OAuth Routes
**File**: `frontend/app/api/calendar/google/route.ts`
- GET: Initiate Google OAuth with calendar scopes
- Scopes: `https://www.googleapis.com/auth/calendar.readonly https://www.googleapis.com/auth/calendar.events.readonly`

**File**: `frontend/app/api/calendar/google/callback/route.ts`
- GET: Handle OAuth callback, exchange code for tokens, store in DB

### 4. Microsoft Outlook OAuth Routes
**File**: `frontend/app/api/calendar/microsoft/route.ts`
- GET: Initiate Microsoft OAuth with calendar scopes
- Scopes: `Calendars.Read offline_access`

**File**: `frontend/app/api/calendar/microsoft/callback/route.ts`
- GET: Handle OAuth callback, exchange code for tokens, store in DB

### 5. Calendar Connection Management
**File**: `frontend/app/api/calendar/connections/route.ts`
- GET: List user's connected calendars
- DELETE: Disconnect a calendar

**File**: `frontend/app/api/calendar/events/route.ts`
- GET: Fetch events from connected calendars for a date range
- Queries both Google and Microsoft APIs using stored tokens
- Handles token refresh automatically

### 6. Calendar Service
**File**: `frontend/lib/services/calendar-service.ts`
- `getGoogleCalendarEvents(accessToken, timeMin, timeMax)` - Fetch Google Calendar events
- `getMicrosoftCalendarEvents(accessToken, timeMin, timeMax)` - Fetch Outlook events
- `refreshGoogleToken(refreshToken)` - Refresh expired Google tokens
- `refreshMicrosoftToken(refreshToken)` - Refresh expired Microsoft tokens
- `getUserConnections(userId)` - Get all connected calendars for a user

### 7. Calendar Integrations Settings Page (Both Locations)
**File**: `frontend/app/settings/integrations/page.tsx`
- Server component that fetches connected calendars

**File**: `frontend/app/settings/integrations/integrations-client.tsx`
- Client component with:
  - Google Calendar connection card (Connect/Disconnect button)
  - Microsoft Outlook connection card (Connect/Disconnect button)
  - List of connected calendars with status
  - Sync toggle for each connection

**File**: `frontend/app/staffs/settings/integrations/page.tsx`
- Staff version of integrations page (same component, different layout)

**File**: `frontend/app/staffs/settings/integrations/integrations-client.tsx`
- Reuses the same integrations client component

### 8. Update Upcoming Tasks Calendar

## Implementation Steps

### Step 1: Database & Config (5 min)
1. Add `calendarConnections` to schema.ts
2. Add environment variables to .env

### Step 2: Google Calendar OAuth (15 min)
1. Create `/api/calendar/google/route.ts` - redirect to Google OAuth
2. Create `/api/calendar/google/callback/route.ts` - handle callback
3. Google OAuth URL: `https://accounts.google.com/o/oauth2/v2/auth`
4. Token exchange: `https://oauth2.googleapis.com/token`

### Step 3: Microsoft Outlook OAuth (15 min)
1. Create `/api/calendar/microsoft/route.ts` - redirect to Microsoft OAuth
2. Create `/api/calendar/microsoft/callback/route.ts` - handle callback
3. Microsoft OAuth URL: `https://login.microsoftonline.com/common/oauth2/v2.0/authorize`
4. Token exchange: `https://login.microsoftonline.com/common/oauth2/v2.0/token`

### Step 4: Calendar Service (10 min)
1. Create `calendar-service.ts` with API helpers
2. Google Calendar API: `https://www.googleapis.com/calendar/v3/calendars/primary/events`
3. Microsoft Graph API: `https://graph.microsoft.com/v1.0/me/calendarView`

### Step 5: Connection Management API (10 min)
1. Create `/api/calendar/connections/route.ts`
2. Create `/api/calendar/events/route.ts`

### Step 6: Settings UI (15 min)
1. Create integrations settings page
2. Add Google/Outlook connection cards
3. Show connected status and disconnect option

### Step 7: Update Calendar UI (15 min)
1. Modify upcoming-tasks-client.tsx to fetch external events
2. Display events with color coding
3. Merge task events and calendar events in the grid
