# Plan: Remove WebSocket (Socket.IO) Except Notifications

## Objective
Strip all Socket.IO functionality from the application except real-time notifications. Replace all real-time data updates with REST APIs + polling/invalidation.

---

## Architecture Overview

**Current state**: Two Socket.IO servers (main API on :4000, chat server on :4001) handling 50+ event types across tasks, chat, meetings, presence, sessions, files, teams, projects, clients, and notifications.

**Target state**: One minimal Socket.IO connection on the main API server handling ONLY the `notification` event. All other functionality uses REST APIs.

**Key decision**: The frontend `getSocketIO()` singleton is kept but stripped to ~60 lines (auth + notification listener only). This avoids changing the `use-notifications.ts` hook which already works correctly.
