== System Architecture Overview

MyWorkSpace is a multi-tenant SaaS platform with an enterprise AI agent system.

=== Architecture Principles

* Clean Architecture with domain-driven organization
* SOLID principles throughout
* Dependency injection via constructor injection
* Repository pattern for data access
* CQRS where appropriate
* Event-driven communication via EventEmitter
* Multi-layer caching (L1 NodeCache, L2 Redis)
* Provider-agnostic AI with intelligent routing

=== Flow: Standard Agent Run

. User sends message → `POST /api/ai/agent/chat`
. Authenticate → Resolve org context
. Load session from memory manager
. Ingest message (normalize, detect language, extract intent/entities)
. Build system prompt (org context + RAG + memories + tools)
. Execute reasoning loop (plan → think → call tools → respond)
. Store conversation in session
. Update episodic memory (enhanced)
. Return formatted response

=== Flow: Multi-Agent Orchestration

. User sends message
. Orchestrator calls PlannerAgent.createPlan()
. Plan decomposed into sequential/parallel tasks
. Task graph resolved (dependency-aware)
. Tasks executed by specialized agents (executor, researcher, critic)
. Quality gate evaluates each result
. Writer agent synthesizes final response
. Reflection performed on complex plans

=== Model Router Strategies

* `manual` - Use configured provider directly
* `auto` - Intelligent provider selection
* `cost_optimized` - Prefer cheapest capable provider
* `latency_optimized` - Prefer fastest provider
* `capability_routed` - Route by required capabilities (vision, embeddings, etc.)

```
                                                          │
User Query → Embed → Vector Search → Hybrid Rank → Context → Prompt
```

== Security Architecture

=== Layers

. **Network**: TLS termination (Caddy), rate limiting, DDoS protection
. **Auth**: JWE-encrypted cookies, Bearer tokens, OAuth providers
. **Authorization**: Casbin RBAC/ABAC policies, org-scoped data isolation
. **Input**: Helmet headers, CSRF tokens, input sanitization middleware
. **AI**: PromptGuard injection detection, dangerous operation approval
. **Audit**: Structured logging with correlation IDs, AI interaction logs

=== PromptGuard Threat Detection

* System prompt override attempts
* DAN/jailbreak patterns
* Instruction extraction attempts
* Role-play exploitations
* Encoding/token smuggling attacks
* Multi-step manipulation
* Delimiter breaking

=== Kubernetes Configuration

* **Namespaces**: Production, Staging, Development
* **Resource Limits**: CPU/Memory requests and limits
* **HPA**: Horizontal Pod Autoscaler (CPU > 70%)
* **PDB**: PodDisruptionBudget (min 2 available)
* **NetworkPolicies**: Pod-to-pod isolation
* **Secrets**: External secrets via SealedSecrets or External Secrets Operator

== Monitoring & Observability

* Prometheus metrics at `/metrics`
* Health check at `/api/health` (MongoDB, Redis status)
* Pino structured JSON logging
* Correlation IDs via `x-request-id`
* Sentry error tracking
* Custom dashboards:
  * API latency p50/p95/p99
  * Token usage and cost by provider
  * Tool usage frequency and latency
  * Memory utilization
  * Active users and sessions

== Performance Considerations

* Multi-layer caching with background refresh
* Connection pooling (Mongoose, Redis)
* Parallel startup initialization
* Context compression at threshold
* Streaming responses for real-time UX
* Reservoir-sampled histograms (bounded memory)
* Lazy provider initialization
* In-memory caches for auth hot path (60s TTL)
